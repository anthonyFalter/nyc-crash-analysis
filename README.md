# NYC Crash Data Preprocessing

This repository contains a simple preprocessing pipeline for the NYC Motor Vehicle Collisions dataset.

## Files

- `preprocessing.ipynb` - notebook with data loading, cleaning, normalization, and export steps
- `dataset/Motor_Vehicle_Collisions_-_Crashes.csv` - raw source data
- `dataset/cleaned_crash_data.csv` - cleaned output generated by the preprocessing phase

## Purpose

Clean and standardize the NYC crash dataset for analysis by:

- normalizing column names to `snake_case`
- combining `crash_date` and `crash_time` into `crash_datetime`
- cleaning street name fields
- normalizing `zip_code`
- grouping vehicle type codes into higher-level categories
- replacing invalid latitude/longitude values with `NaN`
- standardizing contributing factor labels

## Usage

1. Install dependencies:
   ```bash
   pip install pandas numpy matplotlib seaborn
   ```
2. Open and run `preprocessing.ipynb`.
3. The notebook outputs `dataset/cleaned_crash_data.csv`.
4. Figures are saved to `/fig/` folder, used for verifying data points.

## Notes

- The notebook drops extra vehicle and factor columns not needed for the core analysis.
- `primary_street` is currently derived from `on_street_name` with fallback to `off_street_name`.
- Some vehicle type mapping is heuristic and may classify uncommon or ambiguous entries as `OTHER/UNCOMMON`.

## Data Source

Original dataset: NYC Open Data — Motor Vehicle Collisions (`Motor_Vehicle_Collisions_-_Crashes.csv`)
Reference: https://catalog.data.gov/dataset/motor-vehicle-collisions-crashes
